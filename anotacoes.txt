Capítulo 26 Aula 02 – Versão mobile first da tela de login
--------------------------------------------

* box-sizing: `border-box` -> faz com que a borda de um elemento textual seja criada automaticamente de acordo com as configuracoes de largura

* RELEMBRANDO: Existem diversas formas de se centralizar uma div verticalmente e horizonta;mente caso `caixa` esteja configurada a altura e a largura pra ser responsiva a 100% da tela. Mas a forma tradicional é:

 1- colocar a position da caixa pai em : relative

 2- colocar a posição da caixa filha em absoluta.

 3- definir na caixa filha top e left em 50%

 4- definir a transação da div para o centro da tela, para isso os dois valores devem ser configurados em -50%.
    
     EX: transform: translate(-50%, -50%)

* overflow: hidden -> coloca tudo o que está passando para fora da caixa dentro dela forçadamente.
   -aplica-se na classe pai.

* background-position -> define qual parte da imagem de fundo fique aparecendo, caso a caixa que ele foi definida for menor que a imagem, recebe dois valores (ex: left bottom).

* box-shadow: 0px 0px 10px black; -> tem os dois primeiros valos que definem os lados que vão ter o deslocamento da sombra. (Nesse caso nenhum lado). tem 10 de espalhamento e a cor foi configurada em black.
  - necessário definir a transparencia para não ficar muito forte a sombra (geralmente a mais ou menos pela metade).

Capítulo 26 Aula 03 – Outras Media Queries para o projeto
--------------------------------------------

* como na metodologia de desenvolvimento mobile-first é desenvolvido toda estrutura do site visando smartphones, todo o resto da configuração de media-Queries é feito somente as configurações que vao ser para telas diferentes. é colocado somente o código adicional, não é necessário refazer o código todo.

* para inverter de lugar duas divs , basta utilizar a propriedade css 'float:' com o valor 'left' ou 'right' para esquerda/direita,
   - lembrando que as duas caixas devem estar definidas em 'display: block'.

* transition: é usado para dar animação na imagem de fundo de acordo com que ela vai sendo adaptada com a mudança de tela.

  - 'transition-timing-function: ease': faz com que essa animação ganhe uma suavisada para se tornar mais perceptível.

Capítulo 26 Aula 04 – Criando o formulário de login
--------------------------------------------

* Em formulários, a configuração AUTOCOMPLETE serve além de ajudar em completar o que vai ser escrito ele ajudar tambem a validar se o que está sendo escrito está dentro de um padrão de email ou nome, ou o que quer que recebe no seu valor.
  
   - vai dentro do input desejado. 

   - Para ele funcionar nos inputs O AUTOCOMPLETE deve estar ligado (on), na tag principal (form).

   - current-password é um valor melhor para se colocar no atributo 'Autocomplete' para fazer um melhor auxilio no preenchimento de senha do usuário.

* 'required' é um atributo que quando é adicionado em um input de formulário o torna obrigatório para preenchimento.

* 'maxlength=30' torna um input de no maximo 30 caracteres.

* Da mesma forma, existe o 'minlength=8, que define a entrada mínima de caracteres em um input

Capítulo 26 Aula 05 – Aplicando estilo ao formulário de login
--------------------------------------------

* Para criar um seletor css exclusivo para um botão submit(enviar), que fica dentro de um form fica assim o caminho do seletor desta forma:

 form > input(type=submit) {
   ...
 }

 * Para criar um seletor que "pega" um link que está dentro de uma classe botão dentro de um form ficaria assim o caminho: 

 form > a.botao {
   ...
 }
